# AI Agent 模块规划方案

## Summary

基于 `ui-design.md` 中 `4.8 AI Agent` 的页面目标，本次规划将 `AI Agent` 从当前单一占位页，扩展为一个更符合现代化智能体工作台的模块域。规划目标不是一次性做成完整生产级 Agent 平台，而是先定义一个“页面结构清晰、路由完整、最小闭环成立”的原型方案。

本次建议将 `AI Agent` 设计为：

- 1 个父级模块入口

- 3 个可见子页面

- 1 个隐藏详情页

通过这套结构，覆盖“选择智能体 -> 发起运行 -> 查看运行详情/日志 -> 管理配置”的最小闭环，并尽量与当前项目中知识库域的子路由组织方式保持一致。

## Current State Analysis

### 现有页面

- 当前 `AI Agent` 页面只有一个文件：`src/views/agent/index.vue`

- 页面内容仍是占位实现：
  - 左侧：静态智能体列表

  - 顶部：静态标签页

  - 中部：4 张流程能力占位卡片

  - 底部：空调用日志

- 当前实现没有：
  - 子路由

  - 运行任务页

  - 详情页

  - 配置页

  - 测试页

### 现有路由

- 当前生成路由中只有一个一级入口：
  - `agent -> /agent`

- 该路由为单页面入口，尚未形成模块域

### 设计文档约束

`ui-design.md` 中 `4.8 AI Agent` 明确了以下核心结构：

- 左侧边栏选择智能体

- 顶部标签区：运行中、详情、版本、性能评测、配置、测试

- 主内容区：输入理解 / 空间分析 / 智能推理 / 结果输出

- 底部：调用日志

`ui-design.md` 没有规定必须做成多页，但从现代化智能体模块的产品体验出发，如果仍保持在一个页面中，信息密度会迅速失控，不利于扩展，也很难形成“导航内的模块闭环”。

### 与当前项目的一致性

- 当前项目已经开始把复杂模块拆成“父模块 + 子路由域”的方式组织

- 知识库已完成从单页向子路由域的规划和实现思路沉淀，可作为 `AI Agent` 的参考模式

- 项目整体采用深色后台壳，适合做“现代控制台式智能体工作台”

## Assumptions & Decisions

### 已确认目标

- 目标是“规划 AI Agent 模块可以怎么设计”

- 需要参考现代化智能体模块设计

- 需要规划页面和子页面

### 本次规划边界

- 只做规划，不做实现

- 优先规划“最小完整闭环”

- 不追求一次性覆盖完整企业级 Agent 平台能力

### 采用的产品决策

- `AI Agent` 保持一级导航入口不变

- 将 `AI Agent` 扩展为模块父路由，而不是继续停留在单页

- 顶部原设计中的“运行中 / 详情 / 版本 / 性能评测 / 配置 / 测试”，不全部做成平级菜单页

- 其中：
  - `运行中` 和 `详情` 合并为更现代的“运行中心 + 任务详情”关系

  - `版本` 与 `性能评测` 在 MVP 中先降级为智能体详情页中的区块，而不是独立一级页

  - `配置` 和 `测试` 保留为独立子页面

### 最小完整闭环定义

最小闭环应至少能串起以下流程：

- 选择一个智能体

- 发起一次运行任务

- 在运行页面看到任务状态和流程阶段

- 进入详情页查看日志、步骤、引用与结果

- 进入配置页调整该智能体配置

- 进入测试页做 Prompt / 输入测试

## Proposed Changes

### 1. 将 AI Agent 从单页改为父级模块域

#### 目标

保留一级导航 `AI Agent`，但路由结构改为父级承载型模块。

#### 建议路由树

```text
/agent
├─ /agent/workbench        智能体工作台
├─ /agent/config           智能体配置
├─ /agent/test             智能体测试
└─ /agent/task-detail      任务详情（隐藏）
```

#### 设计原因

- 现代智能体模块通常不是单一页面，而是“工作台 + 配置 + 调试/测试 + 任务详情”组合

- 这样比直接把所有标签都塞在一个页面里更清晰

- 同时仍能与设计文档中“顶部标签区”的信息架构保持一致，只是从视觉标签提升为模块子页

### 2. 页面 1：智能体工作台 `workbench`

#### 页面定位

作为 `AI Agent` 模块的主页面，对应设计文档中的主骨架页。

#### 页面结构

- 左侧：智能体列表
  - 地理分析助手

  - 机动规划助手

  - 渡河保障助手

  - 影像识别助手

- 顶部：当前智能体概览区
  - 智能体名称

  - 状态

  - 最近运行信息

  - 主按钮：开始运行

- 中部：能力流程卡片
  - 输入理解

  - 空间分析

  - 智能推理

  - 结果输出

- 右侧或下侧：运行面板
  - 输入参数

  - 推荐模板

  - 任务上下文

- 底部：最近调用日志

#### 最小交互

- 选择左侧智能体后，主区内容切换

- 点击“开始运行”后，生成一条前端任务记录

- 流程卡片显示运行状态：等待 / 运行中 / 完成

- 最近调用日志出现一条最新任务

- 点击某条日志进入 `任务详情`

#### 为什么必须存在

这是设计文档中的主页面骨架，也是用户进入 Agent 模块后的第一视角页面。

### 3. 页面 2：智能体配置页 `config`

#### 页面定位

承载设计文档中“配置”标签的页面化版本。

#### 页面结构

- 左侧：智能体列表或当前智能体切换器

- 右侧：当前智能体配置表单
  - 系统提示词

  - 默认模型

  - 工具开关

  - 输出格式

  - 超时/重试策略

  - 可见性/发布状态

#### 最小交互

- 切换智能体后加载对应配置

- 修改配置并保存

- 保存成功后给出轻提示

#### 为什么放入 MVP

现代化智能体模块如果只有运行页面，没有配置页，会显得“不可运营、不可维护”。

### 4. 页面 3：智能体测试页 `test`

#### 页面定位

承载设计文档中“测试”标签的页面化版本，用于调 Prompt、调输入、看响应。

#### 页面结构

- 左侧：测试输入区
  - 问题输入

  - 场景模板

  - 测试参数

- 右侧：测试结果区
  - 响应结果

  - 工具调用步骤

  - 引用知识/数据

  - 耗时/状态

#### 最小交互

- 输入测试文本后执行

- 返回一段模拟输出

- 展示简化版调用步骤与引用结果

#### 为什么放入 MVP

对于智能体模块来说，“运行”是面向业务，“测试”是面向调试与迭代，两者都属于核心能力。

### 5. 隐藏页：任务详情 `task-detail`

#### 页面定位

承载设计文档中“详情 / 运行中 / 调用日志”的深层查看页。

#### 页面结构

- 头部：任务状态、发起时间、执行人、返回

- 主区：
  - 输入内容

  - 任务上下文

  - 步骤时间线

  - 工具调用明细

  - 输出结果

  - 引用来源

#### 最小交互

- 从工作台日志点击进入

- 显示执行过程与结果

- 支持“重新运行”

#### 为什么不放进菜单

它应是从日志和任务列表下钻进入的详情页，不应作为用户常驻导航入口。

## 信息架构建议

### 一级导航

- 保持现有：`AI Agent`

### 二级导航

- `工作台`

- `配置`

- `测试`

### 隐藏页

- `任务详情`

### 顶部标签如何处理

设计文档中的：

- 运行中

- 详情

- 版本

- 性能评测

- 配置

- 测试

建议在 MVP 中转译为：

- `工作台`：承载“运行中”的主场景

- `任务详情`：承载“详情”

- `配置`：独立子页

- `测试`：独立子页

- `版本`：放入配置页右上角或详情页区块

- `性能评测`：放入测试页下方或配置页二级区块

这样能在不做过多页面的前提下，保留设计文档的语义。

## Assumptions & Decisions

### 明确采用

- 采用“父级路由 + 子页面”方案

- 采用 3 个可见子页 + 1 个隐藏详情页

- 保持与知识库域一致的模块组织方式

### 明确不做

- 不在 MVP 中拆出“版本”独立路由

- 不在 MVP 中拆出“性能评测”独立路由

- 不做复杂流程编排器画布

- 不做真实 LLM 调用

- 不做真实工具链执行

### 视觉方向

- 页面风格建议为“深色运营控制台 + 智能体工作台”

- 左侧智能体列表像“助手切换器”

- 中部流程卡片像“能力链路”

- 底部日志像“运行审计流”

## Verification

### 规划验收标准

- `AI Agent` 不再只是一个单页占位模块

- 具备清晰的父级模块域与子页面结构

- 路由设计能支撑最小完整闭环

- 页面划分与 `ui-design.md` 的结构语义一致

- 页面数量控制在 MVP 可实现范围内，不过度拆分

### 实施后应验证

- 侧边菜单进入 `AI Agent` 后默认跳转 `工作台`

- 能在二级导航切换 `工作台 / 配置 / 测试`

- 从工作台日志进入 `任务详情`

- `任务详情` 不出现在菜单中，但能保持菜单高亮
