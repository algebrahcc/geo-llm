# Frontend Design

This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

## Design Thinking

Before coding, understand the context and commit to a bold aesthetic direction:

- Purpose: What problem does this interface solve? Who uses it?
- Tone: Pick a clear direction (brutally minimal, editorial, luxury, playful, brutalist, retro-futuristic, etc.).
- Constraints: Framework, performance, accessibility, platform.
- Differentiation: What makes it unforgettable?

CRITICAL: Choose a clear conceptual direction and execute it with precision.

## Frontend Aesthetics Guidelines

Focus on:

- Typography: Use distinctive font choices; avoid generic defaults.
- Color & theme: Commit to a cohesive palette; use CSS variables.
- Motion: Prefer a few high-impact interactions over scattered micro-effects.
- Spatial composition: Use intentional layout decisions (asymmetry, overlap, controlled density).
- Backgrounds & detail: Create atmosphere (texture, noise, gradients) when it fits the concept.

Avoid predictable patterns and overused “AI” aesthetics. Match implementation complexity to the vision.
