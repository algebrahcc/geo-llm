## 2. 需要新增的 7 个页面(+ 1 个子页)

> 全 Mock 即可,只做形态。所有"AI 推理"均用 `setTimeout` + 假数据演示流程。

### 2.1 模型管理 → A-5、A-6

- 模型列表(预训练 / 微调中 / 已部署,带状态徽章)
- 版本对比(指标卡:准确率 / 召回 / F1)
- 训练任务进度条 + 训练曲线(ECharts 折线)
- 参数权重滑杆(关键交通节点 / 障碍物识别 / 楼宇夺控 三档)
- 评估报告导出(占位)

### 2.2 数据治理 → A-4

- 清洗任务队列(状态:待处理 / 进行中 / 已完成 / 异常)
- 标准化作业:CRS 转换、字段映射、格式转换(每种 1 个 step 卡片)
- 质检报告(完整度 / 时效性 / 准确性雷达)

### 2.3 影像分析 → A-7

- 影像上传(拖拽)
- 分析任务下发下拉(进攻路线 / 防御工事 / 交通阻断)
- 结果叠加(在 OL 底图上画线/多边形 + 文字标注)
- 缩略图列表(可点击切换)

### 2.4 障碍物识别 → D-1 ~ D-4、D-6

- 图像上传(支持 JPEG/PNG)
- 5 类别下拉多选:**路障 / 地雷 / 反坦克壕 / 伪装网 / 障碍物堆叠**
- 画框标注(每框带类别 + 置信度,不同色)
- 识别率统计 KPI
- 分布热图(简单方格密度)

### 2.5 校验工作台 → A-9、D-5

- 待校验队列(来源:模型输出 / 识别结果 / 抽取参数)
- 原文-结果对照(左原文,右模型输出)
- 操作:批准 / 驳回 / 修订 / 批注
- 校验通过率统计

### 2.6 场景模拟 → A-8(前段)

- 场景模板库(兵力部署 / 气象 / 时段 三类)
- 参数表单(可调权重、滑杆)
- 推演时间轴 + 简易可视化(点位变化)

### 2.7 评估报告 → A-8(后段)

- 报告列表(按场景/时间筛选)
- 报告预览(可导出 PDF 占位按钮)
- 与规则命中率 / 校验通过率 联动

### 2.8 规则配置(子页)→ K-5

- 规则定义表单(条件 + 动作,JSON 编辑)
- 命中预览(输入示例 → 命中结果)
- 版本管理(启用 / 灰度 / 历史)

## 3. 已有 5 个页面需要补的功能

### 3.1 渡河保障 `river` → K-1 ~ K-9(全 9 项集中点)

**最关键模块,9 步流水线全要做**:

1. 知识库检索(K-1)
2. 意图解析(K-2)
3. **参数抽取 + 准确率 badge(≥75%,K-3)**
4. 方案生成 + **多方案对比**(K-4)
5. **规则验证面板(K-5)**
6. 文本处理流水线(K-6)
7. 水域要素图像识别(K-7)
8. 跨模态融合(K-8)
9. **耗时计时器(≤1h,K-9)**

**实现方式**:左侧 step rail(9 个圆点 + 连线),右侧详情面板(随 step 切换)。

### 3.2 机动路线规划 `planning` → A-6、A-7 衍生

- NL 指令解析面板(支持输入"从 A 到 B 避开桥梁")
- 路线权重配置(关键交通节点优先 / 时间最短 / 隐蔽性优先)

### 3.3 地理知识库 `knowledge` → K-1、K-6

- 向量检索可视化(命中片段高亮 + 相似度分)
- 反馈回路(好 / 差按钮,统计命中率)
- 与渡河/规划模块的关联引用

### 3.4 智能体工作台 `agent`

- **工具调用链可视化**(LLM → 工具 → 结果 三段式,带耗时)
- **模型绑定状态指示**(7 个智能体 × 已绑定模型)
- 与 7 类智能体(geo-analysis / mobility-planning / river-crossing / image-recognition / building-control / knowledge / planning)的关联展示

### 3.5 统计大屏 `screen` → A-5、K-3、K-5、D-3

**新增 4 个 KPI + 1 个雷达轴**:

- KPI:模型准确率(%)
- KPI:方案生成平均耗时(min)
- KPI:规则命中率(%)
- KPI:障碍物识别率(%)
- 雷达轴加:校验通过率

## 6. 验收标准(原型)

每完成一个页面/补全,需同时满足:

1. 页面在 Catalog / 任务链路 / 大屏中至少 1 处可见入口
2. "AI 推理"走 `setTimeout(800~1500ms)` + 预设假数据,**禁止使用同步 `Promise.resolve()`**
3. 关键指标数字可从 mock 配置文件读出,方便统一调整
4. 与已有 8 模块的视觉风格(深色玻璃态 + 蓝青配色)保持一致
