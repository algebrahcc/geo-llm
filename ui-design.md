# Geo-LLM 辅助决策原型系统｜界面设计文档（初稿）

## 1. 目标与范围

本文件仅描述界面信息架构、页面布局、组件与交互、状态规范。用于指导前端原型实现与后续迭代评审。

适用页面（按草图）：

- 统计大屏
- Web球
- 数据目录
- 楼宇夺控
- 渡河保障方案
- 机动路线规划
- 知识库
- AI Agent
- 全局：顶栏、侧栏、通知、管理员入口

## 2. 全局界面规范

### 2.1 布局框架

- 顶栏（Header）：系统名 + 一级导航 + 右侧（通知、用户/管理员）
- 内容区（Content）：按页面采用「左侧面板 + 主视图区（地图/图表/表格）+ 右侧面板」三栏结构
- 统一留白：外层容器 padding 16；卡片间距 12～16；面板内控件间距 8～12
- 响应式：
  - > = 1200：三栏完整展示
  - 768～1199：右侧面板可折叠为抽屉
  - < 768：左右面板均抽屉化，主视图优先

### 2.2 视觉层级

- 页面标题：18～20px，半粗
- 区块标题（卡片标题）：14～16px
- 正文与表格：12～14px
- 关键指标：突出数字（20～28px）+ 次级说明（12px）

### 2.3 组件规范（通用）

- 按钮：
  - 主按钮：触发核心动作（智能分析/智能规划/开始 Agent）
  - 次按钮：切换方案/切换路线/导出/查看详情
- 表单：
  - 必填项标识
  - 校验提示：靠近字段，避免弹窗打断
  - 提交态：按钮 loading + 禁用重复提交
- 列表/表格：
  - 支持关键词搜索、筛选、分页
  - 操作列集中：预览/编辑/权限/发布/删除
- 状态规范：
  - Loading：骨架屏（地图/图表）或局部加载
  - Empty：空状态图 + 引导动作（去创建任务/去入库数据）
  - Error：错误提示 + 重试按钮 +（可选）查看日志/详情

### 2.4 简化版 UI/UX 方案（先跑起来）

定位：先用一套“干净、稳、易实现”的样式跑通全链路，后续再逐步增强科技感与微交互。

#### 2.4.1 风格原则

- 地图优先：Web球/路线/方案页以主视图为中心，左右面板可折叠，避免遮挡
- 信息可扫读：表格/卡片对齐统一，默认展示结论与关键指标
- 颜色克制：最多 1 个主色 + 3 个语义色，避免过度装饰

#### 2.4.2 最小颜色 Token（默认深色）

- `primary`: `#2B6BFF`
- `bg`: `#0B1220`
- `surface`: `#0F1B2D`
- `border`: `rgba(148, 163, 184, 0.16)`
- `text`: `rgba(255, 255, 255, 0.92)`
- `text2`: `rgba(255, 255, 255, 0.72)`
- `success`: `#2EE59D`
- `warning`: `#FBBF24`
- `error`: `#FB7185`

地图叠加（路线/风险先用最少集合）：

- `route`: `#2B6BFF`
- `risk`: `#FB7185`

#### 2.4.3 字体与尺寸

- 字体：系统默认即可（Windows 优先 `Microsoft YaHei`），避免引入额外字体成本
- 字号：标题 18，区块标题 14～16，正文/表格 14，辅助信息 12
- 数字：指标卡数字保持右对齐，避免跳动

#### 2.4.4 间距与圆角（MVP）

- 间距：8 / 16 / 24 三档即可
- 圆角：按钮/输入框 8；卡片/面板 12
- 阴影：默认不使用，弹窗/抽屉用一档轻阴影即可

#### 2.4.5 关键组件统一规则

- 顶栏导航：普通 Tab 样式即可，当前页高亮使用 `primary`
- 卡片：标题行 + 内容区；右上角只放 1～2 个常用操作（例如刷新/更多）
- 表格：紧凑密度；操作列优先收敛为“更多”
- 方案/路线切换：用分段控件或 Tab；切换时只替换右侧内容，不刷新整页
- AI 输出展示：固定三段结构即可（结论 / 依据要点 / 建议动作），引用细节后续再加

#### 2.4.6 动效（先不做复杂）

- 仅保留：面板展开/收起 `180ms`，内容淡入 `120ms`
- 地图叠加更新：直接切换，不做复杂动画

#### 2.4.7 UX（先做最重要的 3 件事）

- 默认只展示：结论 + 关键指标 + 风险 Top3
- 跨页保持任务上下文：任务名/范围/时间/状态（最少字段）
- AI 结果可编辑：生成后允许用户手动改，再保存

## 4. 页面设计说明

## 4.1 大屏

目标：展示任务与系统态势总览，支持从总览下钻到任务详情。

### 4.1.1 页面结构（按草图）

- 顶部指标卡（横排）：
  - 任务总数
  - 在线任务
  - 已完成
  - AI 分析次数
  - 用户数
  - 数据总量
- 左侧竖向卡片：
  - 任务分布概览
  - 任务趋势
  - 系统运行状态
- 中央主视图：二维网格图（态势/热力/网格统计）
- 右侧竖向区域：
  - 热点词云（含按钮：知识库、接口）
  - AI 分析任务排名
  - 任务分析通知
- 右侧外延（可折叠抽屉/侧栏）：热点关键词词库

### 4.1.2 关键交互

- 指标卡点击：跳转到任务列表并携带过滤条件（状态/时间范围）
- 网格图交互：
  - Hover：显示网格统计 tooltip
  - Click：联动右侧词云/排名/通知列表
  - 框选（可选）：限定统计范围
- 热点词云：
  - 点击词：触发知识库检索或任务检索
- 任务分析通知：
  - 点击：打开任务详情（含分析过程与结果）

### 4.1.3 状态与异常

- 无任务：展示空态，引导“创建任务”
- 数据加载慢：网格图骨架屏 + 分模块加载
- 接口失败：右侧通知卡提示“加载失败，重试”

## 4.2 Web球

目标：提供独立的三维地球浏览与基础空间交互能力，并为后续分析/标注/路线与方案结果叠加提供统一载体。

### 4.2.1 页面结构（按草图）

- 主视图区：三维地球（全屏或占据内容区绝大部分）
- 左侧竖向工具条：
  - 地理分析工具（入口按钮，展开二级工具）
  - 常用地图工具（定位/测距/测面/标注/清除，具体以实现为准）
- 右侧竖向工具条：
  - 图层控制（底图/影像/DEM/专题/管网/标注/任务叠加）
  - 视角控制（复位、俯仰、旋转、缩放）
- 悬浮分析面板（可拖拽/可折叠）：
  - 标题：AI 分析助手
  - 内容：提示语 + 快捷动作（示例：开始分析、生成标注、导出截图）
- 底部状态栏（可选）：
  - 经纬度、海拔、视角高度、时间/帧率等

### 4.2.2 关键交互

- 场景浏览：拖拽旋转、滚轮缩放、双击定位
- 图层开关：勾选/取消图层后，地图即时刷新；图层顺序可调整（可选）
- 标注工具：支持点/线/面标注；标注可命名、可编辑、可删除；可保存为任务附件（可选）
- 快捷动作：
  - 截图：将当前视角截图保存到“材料/附件”
  - 定位复位：返回默认视角（中国/任务范围）
- 与其它模块联动（跨页面一致）：
  - 在路线规划/方案页面点击“在球上查看”后跳转本页并自动定位到对应范围/路线

## 4.3 数据目录

目标：统一浏览与管理多源地理数据，支持检索、预览、权限与发布。

### 4.3.1 页面结构（按草图）

- 顶部二级 Tab：总览 / 影像数据 / 数字高程 / 地下管网 / 专题数据 / …
- 左侧分类树：
  - 基础影像、数字高程、倾斜摄影、气象水文、矢量管网、…
- 右侧主区：
  - 顶部：搜索框（关键词）
  - 表格列：数据名称 / 入库时间 / 时相 / 空间范围 / 数据类型 / 管理（操作）

### 4.3.2 操作与弹窗

- 入库（管理员）：弹窗/抽屉表单（名称、类型、时相、范围、来源、标签、权限）
- 预览：打开预览面板（地图预览 + 元数据详情）
- 权限：选择可见角色/用户/项目组
- 发布/下线：二次确认
- 删除：危险操作二次确认（含影响提示）

### 4.3.3 搜索与筛选（建议增强，不改变草图主结构）

- 关键词 + 类型 + 时间（入库/时相）+ 空间范围（框选/行政区）
- 最近使用筛选条件记忆

## 4.4 楼宇夺控

目标：围绕任务场景，融合三维地图/BIM/文本信息，并支持按预设路径进行漫游展示与材料生成。

### 4.4.1 页面结构（按草图）

- 左侧卡片：
  - 任务信息（任务名称/规定/目标）
  - 我方力量对比（我方力量信息 ×3）
  - 战场环境
- 中央：三维地图
- 右侧卡片：
  - 建筑文本信息
  - 内部结构 BIM
  - 周边环境
- 底部：情景漫游面板（时间轴/路径控制/讲解点列表）

### 4.4.2 关键交互

- 卡片与地图联动：点击信息卡中的对象 → 地图定位与高亮
- 漫游面板：
  - 预设路线列表
  - 播放/暂停/跳转关键点
  - 截图并生成“讲解材料”

## 4.5 渡河保障方案

目标：从任务输入 → 智能分析 → 多方案生成与对比 → 输出材料/标注。

### 4.5.1 页面结构（按草图）

- 左侧面板：任务输入
  - 任务名称
  - 任务类型
  - 任务范围
  - 人员装备等信息：部队规模、车辆数量、主要装备、时间要求
  - 按钮：智能分析
- 中央：三维地图
- 右侧面板：
  - 分析结果总览（关键结论/指标）
  - 方案切换：方案一 / 方案二 / 方案三
  - 方案内容（结构化摘要 + 文本说明）
  - 标注/材料区（可导出）
- 底部流程条（横向步骤）：
  - 环境分析 → 面源评估分析 → 知识抽取 → 平行方案设计 → 算法模块 → 风险评估 → 输出方案生成

### 4.5.2 关键交互

- 智能分析：
  - 点击后进入“运行中”状态：流程条逐步点亮
  - 允许暂停/取消（可选）
- 方案切换：
  - 切换时地图图层与标注联动（路线/区域/关键点）
  - 右侧内容随方案切换（指标 + 风险点 + 引用）
- 标注：
  - 地图点击添加标注（点/线/面）
  - 标注与方案绑定，可导出到“材料区”
- 导出（可选）：
  - 方案报告（PDF/Word）、地图截图、标注数据（GeoJSON）

### 4.5.3 内容格式（右侧面板建议）

- 关键指标：时间、资源消耗、风险等级、置信度
- 风险列表：风险点、原因、影响范围、缓解建议
- 引用信息：关联数据集/知识条目（仅展示为“引用来源”）

## 4.6 机动路线规划

目标：输入起终点与约束，生成多条候选路线并对比。

### 4.6.1 页面结构（按草图）

- 左侧面板：路线规划表单
  - 起点
  - 终点
  - 路线偏好/约束（多选）
  - 按钮：智能规划
- 中央：三维地图
- 右侧面板：结果说明 + 候选路线按钮
  - 路线一：用时最短
  - 路线二：距离最短
  - 路线三：成本最小（或风险最低）

### 4.6.2 关键交互

- 起终点输入：支持地图点选/坐标输入/搜索（原型可先点选）
- 约束选择：禁行区避让、坡度限制、道路等级、限高限重等（与草图复选框一致）
- 结果展示：
  - 点击路线按钮：地图高亮当前路线，显示指标（时间/距离/风险/成本）
  - 右侧说明区展示“可通行性结论 + 关键障碍/风险点”

### 4.6.3 状态

- 无可行路线：提示原因（禁行区/约束过严）+ 建议放宽哪些约束

## 4.7 知识库

目标：沉淀分析资料与结构化知识条目，支持导入、检索、分类浏览与管理操作。

### 4.7.1 页面结构（按草图）

- 二级区域（内容区顶部）：
  - 知识分类卡片（示例：长江流域、黄河流域、…）
  - 右侧主操作按钮：导入
- 主区：文件/条目表格
  - 列建议：序号、文件名、大小/KB、来源、录入时间、更新时间、审核人、操作
  - 操作列：查看/编辑/删除（管理员）、下载（可选）

### 4.7.2 关键交互

- 分类卡片：点击卡片过滤表格，高亮当前选中项。
- 导入：点击打开表单（文件选择、分类选择、来源填写等）。
- 表格搜索：按文件名/标签/来源关键词过滤。
- 查看/删除：进入详情页或二次确认删除。

## 4.8 AI Agent

目标：以“选择智能体 + 可视化流程编排 + 调用日志”方式，提供分析规划等能力的统一入口。

### 4.8.1 页面结构（按草图）

- 左侧边栏：
  - 地理分析助手
  - 机动规划助手
  - 渡河保障助手
  - 影像识别助手
- 顶部标签区：运行中、详情、版本、性能评测、配置、测试…
- 主内容区：流程能力卡片（从左到右）
  - 输入理解：自然语言理解、空间实体识别、任务解析…
  - 空间分析：地形分析、栅格分析、矢量分析…
  - 智能推理：任务规划、策略选择、质量评估…
  - 结果输出：可视化展示、报告生成、数据导出…
- 底部：调用日志（记录输入、工具步骤、引用、耗时等）

### 4.8.2 关键交互

- 选择智能体：切换左侧列表刷新主区能力展示。
- 运行触发：在“运行中”页签发起任务，可视化显示步骤进度。
- 结果落地：将分析结果一键应用到方案内容或地图标注。

## 5. 图层与地图通用交互（跨页面一致）

- 图层面板：影像/DEM/管网/专题/标注/方案结果图层
- 地图工具：定位、测距、测面、标注、清除
- 结果叠加：路线（线+点）、风险区（面+配色）、网格热力（网格+tooltip）

## 7. 原型验收清单（仅 UI 维度）

- 导航顺序：大屏 -> Web球 -> 数据目录 -> 楼宇夺控 -> 渡河保障方案 -> 机动路线规划 -> 知识库 -> AI Agent
- 页面布局：每个页面布局与设计文档 4.x 章节描述一致
- 关键交互演示：任务输入、分析流转、多方案/多路线切换、三维浏览、知识导入、Agent 调用
- Loading/Empty/Error 状态呈现正常
